# Task 16 – Incident Response & Security Breach Simulation

## Overview

This project demonstrates the process of incident response by simulating a basic security breach scenario and analyzing system authentication logs to identify suspicious activity. The goal is to understand how security professionals detect, investigate, contain, and remediate security incidents.

## Tools Used

* Kali Linux
* Linux Authentication Logs
* Terminal Commands
* System Log Analysis

## Incident Scenario

A simulated security incident was created by performing multiple failed login attempts on a Linux system. These attempts generated authentication logs that were analyzed to detect suspicious activity.

## Objectives

* Simulate a security breach scenario
* Identify suspicious login activity in system logs
* Classify the security incident
* Contain the threat
* Remove the threat source
* Restore the system to a secure state
* Document the incident timeline
* Recommend preventive security measures

## Steps Performed

1. Simulated repeated failed login attempts.
2. Analyzed authentication logs to detect suspicious activity.
3. Classified the incident based on attack type and severity.
4. Contained the incident by isolating the suspicious account.
5. Removed the threat by deleting the malicious account.
6. Restored system security by updating system packages.
7. Documented the incident timeline and actions taken.
8. Recommended preventive security improvements.

## Final Outcome

This task demonstrates the complete incident response lifecycle including detection, analysis, containment, eradication, recovery, and prevention.
